# OpenClaw 插件编写参数设置完全指南

> 基于 [openclaw/openclaw](https://github.com/openclaw/openclaw) 仓库源码整理，面向自定义开发 + 本地安装场景。

---

## 一、插件的三个核心文件

一个完整的 OpenClaw 插件由 **3 个文件** 组成：

```
my-plugin/
├── package.json              ← npm 包描述（声明入口）
├── openclaw.plugin.json      ← 插件清单（声明 ID + 配置 Schema）
└── index.ts                  ← 插件入口（注册逻辑）
```

下面逐一讲解每个文件的参数。

---

## 二、`package.json` 参数设置

### 2.1 字段总览

| 字段 | 必填？ | 类型 | 说明 |
|------|:------:|------|------|
| `name` | ⚠️ 强烈建议 | `string` | 包名，决定默认 plugin ID（取 unscoped 部分）。不填回退为 `"plugin"` |
| `openclaw.extensions` | ✅ **必填** | `string[]` | 入口文件列表。**缺失或为空数组时安装直接报错** |
| `version` | ❌ 可选 | `string` | 版本号，信息性记录 |
| `description` | ❌ 可选 | `string` | 包描述 |
| `type` | ❌ 可选 | `"module"` | ES module 标记，推荐设为 `"module"` |
| `private` | ❌ 可选 | `boolean` | 不发布到 npm 时设为 `true` |
| `dependencies` | ❌ 可选 | `object` | 插件用到的第三方 npm 依赖 |
| `openclaw.channel` | ❌ 可选 | `object` | 仅 channel 类插件需要（onboarding 目录元数据） |
| `openclaw.install` | ❌ 可选 | `object` | 仅需 npm 发布 / onboarding 安装提示时才需要 |

> **唯一硬性必填项**：`openclaw.extensions`。缺失时安装会报错：
> `package.json missing openclaw.extensions`

### 2.2 本地开发最精简写法

```json
{
  "name": "my-plugin",
  "openclaw": {
    "extensions": ["./index.ts"]
  }
}
```

### 2.3 本地开发推��写法

```json
{
  "name": "my-plugin",
  "version": "0.1.0",
  "description": "My custom OpenClaw plugin",
  "type": "module",
  "private": true,
  "openclaw": {
    "extensions": ["./index.ts"]
  }
}
```

### 2.4 有第三方依赖时

```json
{
  "name": "my-plugin",
  "version": "0.1.0",
  "description": "My custom OpenClaw plugin with deps",
  "type": "module",
  "private": true,
  "dependencies": {
    "zod": "^4.3.6",
    "axios": "^1.7.0"
  },
  "openclaw": {
    "extensions": ["./index.ts"]
  }
}
```

安装后需执行：

```bash
cd my-plugin && pnpm install  # 或 npm install
```

### 2.5 `openclaw.channel` 参数（仅 channel 插件）

只有开发消息渠道（如 Telegram / Matrix 类似物）时才需要：

```json
{
  "openclaw": {
    "extensions": ["./index.ts"],
    "channel": {
      "id": "mychannel",
      "label": "My Channel",
      "selectionLabel": "My Channel (API)",
      "detailLabel": "My Channel Detail",
      "docsPath": "/channels/mychannel",
      "docsLabel": "mychannel",
      "blurb": "A custom messaging channel.",
      "order": 70,
      "aliases": ["mc"],
      "preferOver": [],
      "systemImage": "bubble.left",
      "quickstartAllowFrom": true
    }
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | channel ID |
| `label` | `string` | CLI/UI 显示名 |
| `selectionLabel` | `string` | 选择列表中的标签 |
| `detailLabel` | `string` | UI 详情标签 |
| `docsPath` | `string` | 文档路径 |
| `docsLabel` | `string` | 文档标签 |
| `blurb` | `string` | 简短说明 |
| `order` | `number` | 排序权重 |
| `aliases` | `string[]` | 别名列表 |
| `preferOver` | `string[]` | 优先于哪些 channel |
| `systemImage` | `string` | UI 图标名 |
| `quickstartAllowFrom` | `boolean` | 是否允许快速开始 |

### 2.6 `openclaw.install` 参数（仅 npm 发布）

自定义本地开发**不需要**此字段。仅在发布到 npm、需要 onboarding 自动安装提示时使用：

```json
{
  "openclaw": {
    "install": {
      "npmSpec": "@openclaw/my-plugin",
      "localPath": "extensions/my-plugin",
      "defaultChoice": "npm"
    }
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `npmSpec` | `string` | npm 包名（如 `@openclaw/matrix`） |
| `localPath` | `string` | 本地路径（用于 git 仓库开发安装） |
| `defaultChoice` | `"npm" \| "local"` | onboarding 时默认安装方式 |

## 三、`openclaw.plugin.json` 参数设置

### 3.1 字段总览

| 字段 | 必填？ | 类型 | 说明 |
|------|:------:|------|------|
| `id` | ✅ **必填** | `string` | 插件唯一标识符 |
| `configSchema` | ✅ **必填** | `object` | JSON Schema，即使无配置也必须提供空 Schema |
| `name` | ❌ 可选 | `string` | 插件显示名称 |
| `description` | ❌ 可选 | `string` | 插件简短描述 |
| `version` | ❌ 可选 | `string` | 版本号（信息性） |
| `kind` | ❌ 可选 | `string` | 插件类别，目前仅支持 `"memory"`（独占插槽） |
| `channels` | ❌ 可选 | `string[]` | 注册的 channel ID 列表 |
| `providers` | ❌ 可选 | `string[]` | 注册的 provider ID 列表 |
| `skills` | ❌ 可选 | `string[]` | 要加载的技能目录（相对于插件根目录） |
| `uiHints` | ❌ 可选 | `object` | 配置字段的 UI 渲染提示 |

### 3.2 无配置时的最精简写法

```json
{
  "id": "my-plugin",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

### 3.3 有配置项时的推荐写法

```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "description": "A custom plugin for OpenClaw",
  "version": "0.1.0",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "apiKey": { "type": "string" },
      "region": { "type": "string", "default": "us-east-1" },
      "maxRetries": { "type": "number", "minimum": 0 }
    }
  },
  "uiHints": {
    "apiKey": { "label": "API Key", "sensitive": true, "placeholder": "sk-xxx..." },
    "region": { "label": "Region", "placeholder": "us-east-1" },
    "maxRetries": { "label": "Max Retries", "advanced": true }
  }
}
```

### 3.4 `uiHints` 子字段详解

每个配置字段可设置以下 UI 提示：

| 字段 | 类型 | 说明 |
|------|------|------|
| `label` | `string` | 字段标签（Control UI 中显示） |
| `help` | `string` | 帮助/说明文本 |
| `placeholder` | `string` | 输入框占位文本 |
| `sensitive` | `boolean` | 标记为敏感字段（密码/密钥，UI 中会掩码显示） |
| `advanced` | `boolean` | 标记为高级选项（默认折叠） |
| `tags` | `string[]` | 标签分类 |

### 3.5 Channel 插件的 manifest

注册消息渠道时，需在 `channels` 中声明 channel ID：

```json
{
  "id": "mychannel",
  "channels": ["mychannel"],
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "token": { "type": "string" }
    }
  },
  "uiHints": {
    "token": { "label": "Bot Token", "sensitive": true }
  }
}
```

### 3.6 Memory 插件的 manifest

```json
{
  "id": "my-memory",
  "kind": "memory",
  "configSchema": { "type": "object", "additionalProperties": false, "properties": {} }
}
```

> `kind: "memory"` 的插件是独占类型，需通过 `plugins.slots.memory` 选择激活。

## 四、入口文件 `index.ts` 参数设置

### 4.1 两种导出形式

**形式 A — 导出函数：**

```ts
export default function register(api) {
  // 使用 api 注册功能
}
```

**形式 B — 导出对象（推荐，参数更完整）：**

```ts
const plugin = {
  id: "my-plugin",
  name: "My Plugin",
  description: "A demo plugin",
  version: "0.1.0",
  configSchema: emptyPluginConfigSchema(),
  register(api) {
    // 注册逻辑
  },
};
export default plugin;
```

对象形式的参数：

| 字段 | 必填？ | 类型 | 说明 |
|------|:------:|------|------|
| `id` | ❌ | `string` | 覆盖 manifest 中的 id（不匹配时会有警告） |
| `name` | ❌ | `string` | 显示名称 |
| `description` | ❌ | `string` | 描述 |
| `version` | ❌ | `string` | 版本 |
| `kind` | ❌ | `string` | 插件类别 |
| `configSchema` | ❌ | `object` | 运行时配置校验（与 manifest 中的互补） |
| `register` | ✅ | `function` | 注册函数，接收 `api` 对象 |

### 4.2 `api` 对象完整属性与方法

`register(api)` 收到的 `api` 对象定义如下：

```ts
type OpenClawPluginApi = {
  // ========= 只读属性 =========
  id: string;                              // 插件 ID
  name: string;                            // 插件名称
  version?: string;                        // 插件版本
  description?: string;                    // 插件描述
  source: string;                          // 插件源路径
  config: OpenClawConfig;                  // 当前全局配置
  pluginConfig?: Record<string, unknown>;  // 该插件专属配置 (entries.<id>.config)
  runtime: PluginRuntime;                  // 运行时助手 (TTS/STT 等)
  logger: PluginLogger;                    // 日志工具

  // ========= 注册方法 =========
  registerTool(tool, opts?);               // 注册 Agent 工具
  registerHook(events, handler, opts?);    // 注册事件钩子
  registerHttpRoute(params);               // 注册 HTTP 路由
  registerChannel(registration);           // 注册消息渠道
  registerGatewayMethod(method, handler);  // 注册 Gateway RPC 方法
  registerCli(registrar, opts?);           // 注册 CLI 命令
  registerService(service);                // 注册后台服务
  registerProvider(provider);              // 注册模型供应商
  registerCommand(command);                // 注册自动回复命令
  resolvePath(input);                      // 解析路径

  // ========= 生命周期钩子 =========
  on(hookName, handler, opts?);            // 注册生命周期钩子
};
```

### 4.3 各注册方法的参数详解

#### `registerTool(tool, opts?)`

注册 AI Agent 可调用的工具。

| 参数 | 类型 | 说明 |
|------|------|------|
| `tool` | `AnyAgentTool \| ToolFactory` | 工具定义或工厂函数 |
| `opts.name` | `string` | 工具名称 |
| `opts.optional` | `boolean` | 是否为可选工具（需 allowlist 才加载） |

#### `registerHook(events, handler, opts?)`

注册事件钩子。

| 参数 | 类型 | 说明 |
|------|------|------|
| `events` | `string \| string[]` | 事件名或事件名数组 |
| `handler` | `function` | 钩子处理函数 |
| `opts.name` | `string` | 钩子名称 |
| `opts.description` | `string` | 钩子描述 |
| `opts.priority` | `number` | 执行优先级 |

可用事件名：

```
before_model_resolve     before_prompt_build     before_agent_start
llm_input                llm_output              agent_end
before_compaction        after_compaction         before_reset
message_received         message_sending          message_sent
before_tool_call         after_tool_call          tool_result_persist
before_message_write     session_start            session_end
subagent_spawning        subagent_spawned         subagent_ended
subagent_delivery_target gateway_start            gateway_stop
command:new              ...（及其他命令事件）
```

#### `registerChannel(registration)`

注册消息渠道。

```ts
api.registerChannel({
  plugin: {
    id: "mychannel",
    meta: {
      id: "mychannel",
      label: "My Channel",
      selectionLabel: "My Channel (API)",
      docsPath: "/channels/mychannel",
      blurb: "Custom channel.",
      aliases: ["mc"],
    },
    capabilities: { chatTypes: ["direct"] },
    config: {
      listAccountIds: (cfg) => Object.keys(cfg.channels?.mychannel?.accounts ?? {}),
      resolveAccount: (cfg, accountId) =>
        cfg.channels?.mychannel?.accounts?.[accountId ?? "default"] ?? { accountId },
    },
    outbound: {
      deliveryMode: "direct",
      sendText: async ({ text }) => ({ ok: true }),
    },
    // 可选适配器:
    // setup, security, status, gateway, mentions, threading, streaming, actions, commands
    // onboarding: { configure, configureInteractive, configureWhenConfigured }
  },
  dock: undefined,  // 可选
});
```

#### `registerGatewayMethod(method, handler)`

注册 Gateway RPC 方法。

| 参数 | 类型 | 说明 |
|------|------|------|
| `method` | `string` | 方法名（推荐格式 `pluginId.action`） |
| `handler` | `function` | RPC 处理函数 |

```ts
api.registerGatewayMethod("myplugin.ping", ({ respond }) => {
  respond(true, { ok: true, ts: Date.now() });
});
```

#### `registerCli(registrar, opts?)`

注册自定义 CLI 命令。

| 参数 | 类型 | 说明 |
|------|------|------|
| `registrar` | `({ program }) => void` | Commander 注册函数 |
| `opts.commands` | `string[]` | 命令名列表 |

```ts
api.registerCli(
  ({ program }) => {
    program.command("mycommand").action(() => console.log("Hello"));
  },
  { commands: ["mycommand"] },
);
```

#### `registerCommand(command)`

注册自动回复命令（不经过 AI Agent）。

| 参数 | 类型 | 必填？ | 说明 |
|------|------|:------:|------|
| `name` | `string` | ✅ | 命令名（不带 `/`） |
| `description` | `string` | ✅ | 帮助文本 |
| `handler` | `function` | ✅ | 返回 `{ text: string }` |
| `acceptsArgs` | `boolean` | ❌ | 是否接受参数（默认 `false`） |
| `requireAuth` | `boolean` | ❌ | 是否需要授权（默认 `true`） |

Handler 接收的 context：

| 字段 | 类型 | 说明 |
|------|------|------|
| `senderId` | `string` | 发送者 ID |
| `channel` | `string` | 渠道名 |
| `isAuthorizedSender` | `boolean` | 是否授权用户 |
| `args` | `string` | 命令参数 |
| `commandBody` | `string` | 完整命令文本 |
| `config` | `OpenClawConfig` | 当前配置 |

#### `registerService(service)`

注册后台服务。

| 参数 | 类型 | 必填？ | 说明 |
|------|------|:------:|------|
| `id` | `string` | ✅ | 服务 ID |
| `start` | `(ctx) => void \| Promise<void>` | ✅ | 启动函数 |
| `stop` | `(ctx) => void \| Promise<void>` | ❌ | 停止函数 |

ServiceContext 包含：`config`、`workspaceDir`、`stateDir`、`logger`。

#### `registerProvider(provider)`

注册模型供应商认证。

```ts
api.registerProvider({
  id: "acme",
  label: "AcmeAI",
  auth: [
    {
      id: "oauth",
      label: "OAuth",
      kind: "oauth",
      run: async (ctx) => ({
        profiles: [{ profileId: "acme:default", credential: { ... } }],
        defaultModel: "acme/opus-1",
      }),
    },
  ],
});
```

#### `registerHttpRoute(params)`

注册 Gateway HTTP 路由。

| 参数 | 类型 | 说明 |
|------|------|------|
| `method` | `"GET" \| "POST" \| ...` | HTTP 方法 |
| `path` | `string` | 路由路径 |
| `handler` | `function` | 请求处理函数 |

### 4.4 `logger` 方法

```ts
api.logger.info("informational message");
api.logger.warn("warning message");
api.logger.error("error message");
api.logger.debug?.("debug message");  // debug 可能为 undefined
```

### 4.5 `runtime` 运行时助手

```ts
// TTS - 文本转语音（电话场景）
await api.runtime.tts.textToSpeechTelephony({ text: "Hello", cfg: api.config });

// STT - 语音转文本
await api.runtime.stt.transcribeAudioFile({
  filePath: "/tmp/audio.ogg",
  cfg: api.config,
  mime: "audio/ogg",
});
```

## 五、OpenClaw 配置文件中的插件参数（`openclaw.json`）

### 5.1 `plugins` 完整配置结构

```json5
{
  plugins: {
    enabled: true,                          // 主开关（默认 true）
    allow: ["my-plugin"],                   // 允许列表（可选）
    deny: ["untrusted-plugin"],             // 拒绝列表（deny 优先）
    load: {
      paths: ["/path/to/my-plugin"],        // 额外加载路径
    },
    slots: {
      memory: "memory-core",               // 独占类别插槽
    },
    entries: {
      "my-plugin": {
        enabled: true,                      // 单插件开关
        config: {                           // 传给 api.pluginConfig
          apiKey: "sk-xxx",
          region: "us-east-1",
        },
        apiKey: "convenience-key",          // 便捷 API key（若插件支持）
        env: { MY_VAR: "value" },           // 插件作用域环境变量
      },
    },
    installs: { ... },                      // CLI 自动管理，不建议手写
  },
}
```

### 5.2 `entries.<id>` 参数详解

| 字段 | 类型 | 说明 |
|------|------|------|
| `enabled` | `boolean` | 是否启用（默认 `true`） |
| `config` | `object` | 插件自定义配置（由 manifest 中的 `configSchema` 校验） |
| `apiKey` | `string` | 便捷 API key（支持 SecretRef） |
| `env` | `Record<string, string>` | 注入该插件运行时的环境变量 |

### 5.3 `installs.<id>` 参数详解（CLI 管理，通常不手写）

| 字段 | 类型 | 说明 |
|------|------|------|
| `source` | `"npm" \| "archive" \| "path"` | 安装来源类型 |
| `spec` | `string` | npm 包规格 |
| `sourcePath` | `string` | 原始本地路径 |
| `installPath` | `string` | 安装目录 |
| `version` | `string` | 安装时版本 |
| `resolvedName` | `string` | npm 解析的包名 |
| `resolvedVersion` | `string` | npm 解析的精确版本 |
| `resolvedSpec` | `string` | npm 解析的精确 spec |
| `integrity` | `string` | npm dist 完整性哈希 |
| `shasum` | `string` | npm dist shasum |
| `resolvedAt` | `string` | 解析时间（ISO） |
| `installedAt` | `string` | 安装时间（ISO） |

## 六、本地安装方式

### 开发模式（推荐，修改即时生效）

```bash
openclaw plugins install -l ./my-plugin
# 自动添加到 plugins.load.paths，无需手写 installs
```

### 复制安装

```bash
openclaw plugins install ./my-plugin
# 复制到 ~/.openclaw/extensions/my-plugin/
```

### 手动配置（不用 CLI）

直接编辑 `openclaw.json`，完全跳过 `installs`：

```json5
{
  plugins: {
    load: { paths: ["/home/user/Projects/my-plugin"] },
    entries: { "my-plugin": { enabled: true } },
  },
}
```

## 八、完整最小示例（可直接复制使用）

**`my-plugin/package.json`：**

```json
{
  "name": "my-plugin",
  "version": "0.1.0",
  "type": "module",
  "private": true,
  "openclaw": {
    "extensions": ["./index.ts"]
  }
}
```

**`my-plugin/openclaw.plugin.json`：**

```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "description": "A minimal custom plugin",
  "version": "0.1.0",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

**`my-plugin/index.ts`：**

```ts
export default function register(api) {
  api.logger.info(`${api.name} loaded from ${api.source}`);

  // 注册一个自动回复命令
  api.registerCommand({
    name: "hello",
    description: "Say hello from my plugin",
    handler: () => ({ text: "Hello from My Plugin! 🎉" }),
  });

  // 注册一个后台服务
  api.registerService({
    id: "my-heartbeat",
    start: () => api.logger.info("Heartbeat started"),
    stop: () => api.logger.info("Heartbeat stopped"),
  });

  // 注册一个 Gateway RPC 方法
  api.registerGatewayMethod("myplugin.ping", ({ respond }) => {
    respond(true, { ok: true, timestamp: Date.now() });
  });
}
```

**安装并运行：**

```bash
openclaw plugins install -l ./my-plugin
# 重启 Gateway 即可生效
```

## 九、速查：哪些必填、哪些可选

| 文件 | 必填字段 | 推荐填写 | 可选（按需） |
|------|---------|---------|------------|
| `package.json` | `openclaw.extensions` | `name`、`version`、`type`、`private` | `description`、`dependencies`、`openclaw.channel`、`openclaw.install` |
| `openclaw.plugin.json` | `id`、`configSchema` | `name`、`description`、`version` | `kind`、`channels`、`providers`、`skills`、`uiHints` |
| `index.ts` | 导出函数或含 `register` 的对象 | — | 对象形式中的 `id`、`name`、`configSchema` |
| `openclaw.json`（用户配置） | — | `plugins.entries.<id>.enabled` | `plugins.load.paths`、`entries.<id>.config`、`entries.<id>.env` |
